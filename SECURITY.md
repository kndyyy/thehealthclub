# Security Policy

## Supported Versions

We take security seriously and appreciate your efforts to responsibly disclose vulnerabilities. The following versions of our project are currently supported and will receive security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark:  |
| 0.9.x   | :white_check_mark:  |
| < 0.9   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, we would like to know about it so we can take steps to address it as quickly as possible. Please follow the guidelines below to report the issue:

- **Email**: Please send an email to [security@thehealthclub.com](mailto:security@thehealthclub.com) with the details of the vulnerability.
- **Include**:
  - A detailed description of the vulnerability.
  - Steps to reproduce the issue, if possible.
  - Any potential impact or severity of the vulnerability.
- **Response Time**: We aim to respond to all security vulnerability reports within 48 hours. Once we verify the issue, we will take steps to mitigate it and will provide an update on the status of the fix.
- **Confidentiality**: Please do not disclose the vulnerability publicly until we have addressed it and released an update.

## Our Commitment

We are committed to keeping our community safe and appreciate your efforts in helping us maintain the security of our platform. We will:

- Acknowledge the receipt of your report within 48 hours.
- Work with you to understand the issue and its implications.
- Implement a fix if necessary and release a patch as soon as possible.
- Credit you for the discovery if you wish, once the vulnerability has been resolved.

Thank you for your support and collaboration!
